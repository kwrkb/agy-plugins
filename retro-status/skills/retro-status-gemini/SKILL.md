# retro-status MCP プラグイン（retro_status）使用ガイド

## ツール概要

このプラグインは `retro_status` という単一の MCP ツールを提供する。

```
retro_status(path="<repository_path>", format="text")
```

### パラメータ
- `path` (string, 任意): スキャンするリポジトリのパス。指定がない場合はカレントディレクトリ `.`。
- `format` (string, 任意): `text` または `json`。デフォルトは `text`。

## 呼び出し条件

- ユーザーから「現在のリポジトリのステータスを見せて」「`/status`」「`/retro-status`」「レトロステータスを出して」などの要望があった場合に呼び出す。
- デフォルトの `format` は `text` を使用し、ファミコン風のアスキーアート（AA）を生成する。

## 出力時の最重要ルール

1. **アスキーアートを破壊しない**:
   - `format="text"` で出力されたテキスト（アスキーアート）は、` ``` ` などのコードブロックで囲んでそのまま出力すること。
   - アスキーアート内の空白や枠線文字が崩れないよう、整形や文字幅の改変は一切行ってはならない。
2. **余計な解説を最小限にする**:
   - 出力されたアスキーアート自体が情報を含んでいるため、解説は必要最小限に留める。
   - 「以下はリポジトリのレトロステータス画面です」といった短いイントロダクションのみを添えて提示する。
